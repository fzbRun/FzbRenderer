Scene XML 读取流程（结构化）

下列 HTML 按原流程逐级分解了读取 sceneXML 的步骤与子步骤，便于在文档、页面或开发规范中使用。

  1. scene 会从 sceneXML 中读取全局信息
    1. 读取场景级别（全局）配置：渲染设置、路径、默认参数等。
  2. scene 初始化
    1. 为 scene 添加各种全局信息
    2. 创建 mesh 的描述符集合
    3. 从 sceneXML 中读取数据
      1. 读取相机信息
      2. 读取 material
      3. 根据 material 的 type 创建 shader，并创建 shaderVariant
        1. 根据 material 的属性类型，关闭没有传入的属性的宏（即针对缺省属性关闭对应编译宏）
        2. 根据传入的 vertexFormat 参数和开启的宏，决定 shaderVariant 的 vertexFormat，并与 material 同步。
      4. 读取 mesh 信息（包括光源）
        1. 读取 mesh 的路径，从 .obj 中读取数据（顶点和索引）。读取顶点时读入其 vertexFormat，并与 material 同步。
        2. 识别 mesh 的 materialID，将之与之前读取的 material 关联。
        3. 根据 mesh 的 vertexFormat 将其放入集合中，用于后续压缩数据处理。
  3. 创建顶点和索引缓冲
    1. 每个 vertexFormat 集合分别压缩（即对相同 vertexFormat 的 mesh 的顶点数据进行合并/压缩）
      1. 从 mesh 中读取顶点和索引数据。
      2. 根据当前已读的相同 vertexFormat 的 mesh 的索引数量和所有之前读取的（不同 vertexFormat）的索引数量，为 mesh 的 indexArrayOffset 赋值，为后续渲染 offset 做准备。
      3. 创建一个 mesh 的索引数组副本，根据已读的相同 vertexFormat 的 mesh 的索引数量，为该副本的索引值做偏移（将局部索引映射到压缩数组的全局索引）。
      4. 将该 vertexFormat 的所有顶点数据和偏移后的索引副本数组放入两个大的压缩数组中（一个顶点数组和一个索引数组）。
    2. 将压缩数组传给压缩函数进行压缩（例如顶点量化、索引类型缩减、去重等）。
    3. 将压缩后的顶点数组和索引数组加入两个大的场景顶点和索引数组中
      1. 如果之前的 vertexFormat 的顶点数组的 float 数不能整除当前 vertexFormat，则需要加入一些 padding 以对齐内存布局。
      2. 将整除后的结果加到索引压缩数组的索引值上去（修正索引的全局偏移）。
    4. 最后根据场景顶点数组和索引数组，创建最终的顶点缓冲和索引缓冲（GPU 上传、描述符绑定等）。
